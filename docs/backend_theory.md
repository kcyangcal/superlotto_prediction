## 二、模型原理說明

---

### 2.1 頻率基準（Frequency Baseline）

#### 原理

最簡單的統計推論：**歷史出現越多次的號碼，未來越可能出現**。

```
score(號碼 i) = α × freq_all(i)
              + β × freq_mid(i)    （α > β > γ）
              + γ × freq_short(i)
```

加權融合三個時間視窗（全歷史 50%、中期 30%、短期 20%），讓近期頻率有更高影響力。

#### 適合場景

- 資料量小、模型訓練成本預算低
- 作為所有複雜模型必須超越的基準線（Baseline）
- 快速產出初步推薦（毫秒級）

#### 優缺點

| 優點                          | 缺點                                    |
|-------------------------------|-----------------------------------------|
| 極簡，完全可解釋               | 忽略「號碼組合」與「時序依賴」            |
| 訓練瞬間完成                  | 假設歷史頻率代表未來機率（對隨機資料不成立） |
| 不需要特徵工程                | 所有模型幾乎都應能超越此基準             |

---

### 2.2 蒙地卡羅模擬（Monte Carlo Simulation）

#### 原理

以歷史頻率為**抽樣權重**，模擬大量虛擬開獎（10 萬次），統計每顆球在模擬中的出現頻率。

```python
# 虛擬碼
white_weights = historical_frequency / sum(historical_frequency)
for _ in range(100_000):
    simulated_draw = np.random.choice(47_numbers, size=5, replace=False, p=white_weights)
    count[simulated_draw] += 1
推薦 = top 5 by count
```

#### 為何不同於純頻率排名？

純頻率排名是「確定性計算」；蒙地卡羅在每次模擬中加入**隨機性**，因此可以捕捉到**號碼共現的組合效應**。

例如：號碼 7 和 14 的歷史頻率都中等，但若它們**常常同時出現**，模擬中這個組合的出現頻率會超過兩個獨立頻率的乘積。

#### 優缺點

| 優點                              | 缺點                              |
|-----------------------------------|-----------------------------------|
| 捕捉組合效應（共現模式）           | 結果隨機種子而略有波動             |
| 概念直觀，易於解釋                | 計算量高（10萬次迴圈）             |
| 可調整模擬次數平衡速度與精度       | 本質上仍依賴歷史頻率假設           |

---

### 2.3 馬可夫鏈（Markov Chain）

#### 數學底層

**一階馬可夫性質**：未來狀態只依賴當前狀態，與更遠的過去無關。

建立 **47 × 47 轉移矩陣 T**：

```
T[i][j] = P(號碼 j 在下一期出現 | 號碼 i 在這期出現)
         = count(i 和 j 在相鄰兩期同時出現) / count(i 出現)
```

**預測邏輯**：

```
# 取最後一期 5 顆球的轉移機率之和
combined_prob = Σ T[ball - 1]  for ball in last_draw
推薦 = top 5 by combined_prob
```

#### 正規化（Laplace Smoothing）

若某號碼從未出現過（T[i] 全為 0），使用均等分布填充：`T[i] = 1/47`。這避免了「除以零」問題，是貝氏先驗的一個簡單形式。

#### 適合場景

- 資料中存在「序列依賴性」（第 i 期的號碼影響第 i+1 期）
- 適合作為理解馬可夫性質的教學模型

#### 優缺點

| 優點                              | 缺點                                       |
|-----------------------------------|--------------------------------------------|
| 有理論基礎（馬可夫隨機過程）       | 假設序列依賴性——彩票實際上是獨立事件        |
| 轉移矩陣可視化與分析               | 一階馬可夫忽略「2期前」的影響              |
| 訓練快速（O(n) 歷史資料掃描）     | 矩陣稀疏（大多數 i→j 轉移頻率很低）       |

---

### 2.4 貝氏推論（Bayesian Inference）

#### 數學底層

**貝氏定理**：

```
P(號碼出現 | 觀測) ∝ P(觀測 | 號碼出現) × P(號碼出現)
    後驗機率          似然              先驗機率
```

**本實作的具體設計**：

- **先驗（Prior）**：歷史出現頻率（正規化為機率分布）
  ```
  prior(i) = freq_all(i) / Σ freq_all
  ```

- **似然（Likelihood）**：基於 Gap 值的「overdue」分數
  ```
  # Gap 越大 → 號碼越「過期」→ 似然越高（gambler's fallacy 式假設）
  likelihood(i) = exp(gap[i] - mean(gap)) / Σ exp(gap - mean(gap))
  ```
  注意：`softmax(gap)` 用均值平移避免數值溢位（exp 的數值穩定技巧）。

- **後驗融合**：
  ```
  posterior(i) = prior_weight × prior(i) + likelihood_weight × likelihood(i)
  推薦 = top 5 by posterior
  ```

#### 重要說明

似然函數中使用 Gap 值的假設（號碼「欠」了，所以快要出現）在理論上是**賭徒謬誤（Gambler's Fallacy）**。本模型的學習價值在於理解貝氏框架，而非驗證這個假設。

#### 優缺點

| 優點                              | 缺點                                       |
|-----------------------------------|--------------------------------------------|
| 標準貝氏框架，概念清晰             | 似然設計依賴主觀假設（Gap → 賭徒謬誤）     |
| 可調整先驗/似然權重                | 對真正獨立的隨機資料，後驗不會優於先驗     |
| 計算快速                          | 需要領域知識來設計有意義的似然函數         |

### 2.5 KNN 最近鄰（K-Nearest Neighbors）

#### 數學底層

**核心思想**：「現在的情況」與「歷史上某幾期的情況」最相似，那麼那幾期的「下一期」可能就是我們要預測的。

**特徵向量設計（94 維）**：
```
fv = [gap_1, gap_2, ..., gap_47,       ← 47維 Gap 特徵
      freq_1, freq_2, ..., freq_47]     ← 47維 近期頻率特徵
```

**距離度量**：歐氏距離（Euclidean Distance）。在計算前先對特徵做 z-score 標準化，確保不同量級的特徵具有可比性：

```
fv_scaled = (fv - mean) / std
distance(a, b) = √(Σ (a[i] - b[i])²)
```

**投票機制**：
```
找出 K 個最近鄰（最相似的歷史期）
對每個近鄰，查詢其「下一期」實際開出的號碼
統計這些號碼的出現次數 → 推薦出現最多次的 5 顆
```

#### 優缺點

| 優點                              | 缺點                                         |
|-----------------------------------|--------------------------------------------- |
| 非參數方法，不需要假設分布         | 特徵向量建構耗時（每期都要重算）              |
| 直觀可解釋（可顯示「相似的歷史期」）| 維度詛咒（94 維空間中，「距離」意義稀釋）    |
| 可調整 k 值控制平滑程度           | 若歷史統計規律本身無序，KNN 效果接近隨機     |

### 2.6 決策樹多標籤分類（Decision Tree Multi-Label）

#### 數學底層

決策樹透過反覆選擇**分裂特徵**與**分裂閾值**，將訓練資料分割成越來越純粹的子集。

**不純度衡量（Gini Impurity）**：
```
Gini(S) = 1 - Σ p_c²
```
其中 `p_c` 是集合 S 中類別 c 的比例。分裂時選擇能最大程度降低 Gini Impurity 的特徵和閾值。

**多標籤設計（Multi-Output）**：
```
為每個號碼（1–47）獨立訓練一棵決策樹
問題：「給定特徵向量，號碼 i 在下一期是否出現？」→ 二元分類
預測：取各號碼「出現機率」最高的 5 個
```

**類別不平衡處理**：`class_weight='balanced'` 自動將「未出現（89.4%）」類的損失權重降低，防止模型把所有樣本都預測為「不出現」。

#### 優缺點

| 優點                              | 缺點                                       |
|-----------------------------------|--------------------------------------------|
| 高度可解釋（可視化決策路徑）       | 容易過擬合（需剪枝：max_depth、min_samples） |
| 能捕捉非線性特徵交互               | 單棵樹的預測方差高（不穩定）               |
| `feature_importances_` 量化特徵影響 | 相比 RF，表現通常較差                     |

---

### 2.7 隨機森林（Random Forest）

#### 數學底層

Random Forest 是決策樹的**集成（Ensemble）**版本，基於兩個隨機化策略：

1. **Bagging（Bootstrap Aggregating）**：每棵樹訓練在「有放回抽樣」的子資料集上（Bootstrap Sample），約 63.2% 的原始樣本參與，剩下 36.8% 稱為 Out-of-Bag（OOB）。

2. **特徵隨機選取**：每個分裂點只從隨機選取的 `sqrt(n_features)` 個特徵中選最佳分裂。

**預測**：對每棵樹的預測結果投票（分類）或取均值（迴歸）。

```
RF_prediction = majority_vote(Tree_1, Tree_2, ..., Tree_N)
```

**為何集成有效？**
- 每棵樹的「錯誤方向」是隨機的（不相關誤差）
- 多棵樹投票時，隨機誤差相互抵消
- 集成後的偏差（Bias）略高，但方差（Variance）顯著降低

#### 優缺點

| 優點                              | 缺點                                         |
|-----------------------------------|----------------------------------------------|
| 對過擬合具有顯著抵抗力             | 計算量高（300 棵樹 × 47 個分類器）           |
| 提供可靠的特徵重要性               | 模型不可直接解釋（Black Box）               |
| 無需特徵縮放                      | 訓練時間長（數分鐘）                         |
| 穩定、可重現                      | 對時序模式的捕捉能力有限                     |

---

### 2.8 XGBoost（Extreme Gradient Boosting）

#### 數學底層

與 Random Forest 的「並行集成（Bagging）」不同，XGBoost 是**序列集成（Boosting）**：

每一棵新樹的目標是「修正前一輪的殘差（Residuals）」。

```
Ŷ_m = Ŷ_{m-1} + η × h_m(x)
```

其中 `η` = learning_rate（防止過擬合），`h_m` = 第 m 棵樹（擬合上一輪的殘差）。

**與 RF 最大的結構差異**：

| 特性           | Random Forest           | XGBoost                     |
|----------------|-------------------------|-----------------------------|
| 集成方式       | 並行（Bagging）         | 序列（Boosting）             |
| 每棵樹目標     | 擬合原始標籤            | 擬合前一輪的殘差             |
| 正則化         | 無內建正則化            | L1 + L2 正則化項             |
| 對不平衡資料   | `class_weight='balanced'` | `scale_pos_weight` 精細控制 |

**本專案的不平衡設定**：

```python
scale_pos_weight = 42/5 ≈ 8.4
# 正樣本（出現）的損失函數被放大 8.4 倍
# 這讓模型更「在乎」預測出現的號碼是否正確
```

#### 優缺點

| 優點                              | 缺點                                         |
|-----------------------------------|----------------------------------------------|
| 通常在結構化資料表現最好           | 超參數多，調優複雜                           |
| 精細的不平衡資料處理               | 訓練時間較長                                 |
| 內建正則化（L1/L2）               | 對時序趨勢的建模不如 LSTM                    |
| 可早停（Early Stopping）          | Boosting 序列無法並行（外層 n_jobs=1）       |

### 2.9 遺傳演算法（Genetic Algorithm）

#### 數學底層

遺傳演算法模擬生物進化，透過**選擇、交叉、突變**在解空間中搜尋最優解。

**染色體表示**：一個長度為 5 的整數列表 = 一個號碼組合
```
染色體 = [7, 15, 23, 36, 42]
```

**適應度函數**：
```
f(組合) = freq_weight × Σ normalized_freq(ball)
         + cooccur_weight × Σ cooccurrence(ball_i, ball_j)
```

其中共現矩陣 `cooccur[i][j]` = 號碼 i 和 j 在歷史中同期出現的次數（正規化後）。

**演化運算子**：

| 運算子     | 說明                                              |
|------------|---------------------------------------------------|
| 選擇       | 錦標賽選擇：隨機取 k 個，選 fitness 最高者         |
| 交叉       | 合併父代基因池，隨機選 5 顆生成子代               |
| 突變       | 每顆球以 mutation_rate=15% 的機率替換為其他號碼    |
| 精英保留   | 前 10% 的個體直接複製到下一代（保留最佳解）        |

#### 優缺點

| 優點                              | 缺點                                         |
|-----------------------------------|----------------------------------------------|
| 可搜尋複雜的組合空間               | 結果依賴適應度函數設計（主觀性強）           |
| 能找到頻率+共現的最優組合         | 收斂結果有隨機性（不同隨機種子結果不同）     |
| 算法原理具有廣泛應用價值           | 不適合高維連續最佳化問題                     |

---

### 2.10 集成學習（Ensemble Voting）

#### 原理

不依賴任何單一模型，而是對所有子模型的推薦結果進行**加權投票**：

```
vote_score[號碼 i] = Σ weight[模型] × I(號碼 i 在模型推薦列表中)
最終推薦 = top 5 by vote_score
```

**預設權重設計邏輯**：

| 模型       | 權重 | 理由                               |
|------------|------|------------------------------------|
| 貝氏推論   | 1.2  | 融合先驗與似然，資訊較豐富           |
| KNN        | 1.2  | 基於特徵相似度，考量更多維度         |
| 頻率基準   | 1.0  | 穩定的歷史統計                     |
| 蒙地卡羅   | 1.0  | 模擬隨機性                         |
| 馬可夫鏈   | 1.0  | 序列轉移機率                       |
| 決策樹     | 0.8  | 容易過擬合，略降權重               |
| 遺傳演算法 | 0.8  | 適應度函數主觀性較強               |

| XGBoost      | 慢       | 中等     | 低       | 多     |

### 3.2 理論假設

| 模型         | 核心假設                                    | 彩票適用性 |
|--------------|---------------------------------------------|------------|
| 頻率基準     | 歷史頻率 ≈ 未來機率                         | 低         |
| 蒙地卡羅     | 同上，但考量抽樣不放回                       | 低         |
| 馬可夫鏈     | 一階序列依賴（開獎間有記憶）                 | 低（彩票無記憶） |
| 貝氏推論     | 先驗 × 似然 → 後驗（Gap → 似然合理性存疑） | 低         |
| KNN          | 相似當下狀態 → 相似未來                     | 低         |
| 決策樹/RF    | 特徵→標籤的非線性映射                       | 低         |
| XGBoost      | 同上 + 殘差序列修正                         | 低         |
| 遺傳演算法   | 適應度函數能捕捉「好組合」的本質             | 低         |

> **結論**：所有模型的「彩票適用性」都標記為「低」。這是誠實的評估——
> **彩票的設計本質就是獨立均勻隨機**，任何歷史統計規律都不應對未來有預測力。
> 本專案的學習價值在於「建立評估管線」與「理解各演算法原理」，而非「找到彩票規律」。

### 3.3 適合場景對比（真實 ML 應用）

| 模型         | 真正適合的領域                              |
|--------------|---------------------------------------------|
| 頻率/統計    | 客戶行為分析、庫存預測（頻率有真實意義時）  |
| 馬可夫鏈     | 網頁導航預測、用戶行為序列建模              |
| 貝氏推論     | 醫療診斷、垃圾郵件過濾、A/B 測試分析        |
| KNN          | 推薦系統、異常偵測（特徵設計良好時）        |
| 決策樹       | 信用風險評估（需要可解釋結論時）            |
| 隨機森林     | 通用結構化資料分類（Kaggle 競賽常勝模型）   |

### 2.11 多層感知機 MLP（PyTorch 版）

#### 數學底層

MLP 是最基礎的神經網路形式，由多個**全連接層（Fully Connected Layer）**堆疊而成。

**前向傳播（Forward Pass）**：
```
h¹ = ReLU(W¹x + b¹)          ← 第一隱藏層
h² = ReLU(W²h¹ + b²)         ← 第二隱藏層
ŷ  = sigmoid(W³h² + b³)      ← 輸出層：47 個機率值
```

**損失函數**：BCEWithLogitsLoss（Binary Cross Entropy with Logits）
```
L = -Σ [pos_weight × y·log(σ(ŷ)) + (1-y)·log(1-σ(ŷ))]
pos_weight = 42/5 ≈ 8.4  （補償類別不平衡）
```

**反向傳播（Backpropagation）**：
透過鏈式法則（Chain Rule）計算損失對每個參數的梯度，然後用 Adam 優化器更新：
```
θ ← θ - α × ∇_θ L
```

**BatchNorm 的作用**：對每個 Mini-batch 的激活值做標準化，加速訓練收斂並減少對初始化的依賴。

**Dropout 的作用**：訓練時隨機關閉 30% 的神經元，防止模型過度依賴某幾個特徵（正則化效果）。

**Rolling Update**：
```
partial_fit(new_draw) → 只對最近 100 期資料做 5 個 epoch 的微調
```

#### 優缺點

| 優點                              | 缺點                                         |
|-----------------------------------|----------------------------------------------|
| 能學習特徵間的非線性交互           | 忽略時序順序（特徵是「快照」而非序列）        |
| BatchNorm + Dropout 提供正則化     | 超參數多（層數、神經元數、學習率）           |
| GPU 加速支援（CUDA/MPS）           | 訓練資料量少時容易過擬合                     |
| partial_fit 支援增量學習           | 黑盒，可解釋性低                             |

### 2.12 LSTM 長短期記憶網路（PyTorch 版）

#### 數學底層

LSTM 是專門為**序列資料**設計的神經網路，核心是三個閘門機制：

**遺忘閘（Forget Gate）**：
```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
```
值域 (0, 1)，控制「上一時刻的記憶保留多少」。

**輸入閘（Input Gate）**：
```
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)
```
控制「本時刻的新資訊寫入多少」。

**細胞狀態更新（Cell State Update）**：
```
C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t
```
`C_t` 是跨越長時間序列的「記憶帶」，梯度可以不衰減地流過。

**輸出閘（Output Gate）**：
```
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
h_t = o_t ⊙ tanh(C_t)
```

**本專案的 LSTM 架構**：
```
輸入：(batch, seq_len=20, 47)
  ↓
LSTM(hidden=128, layers=2, dropout=0.3)
  ↓ 取最後時間步 h_T
Linear(128→64) → ReLU → Dropout(0.2)
  ↓
Linear(64→47) → sigmoid → 47個機率值
```

**梯度裁剪（Gradient Clipping）**：LSTM 中可能發生梯度爆炸（Exploding Gradient），透過 `clip_grad_norm(max_norm=1.0)` 防止。

**Rolling Update（Catastrophic Forgetting 的處理）**：
```python
# fine-tuning 時學習率降為 1/10，緩解遺忘問題
lr_finetune = lr_initial × 0.1
partial_fit(new_draws, epochs=5)
```

#### 為何 LSTM 比 MLP 更適合時序問題

| 對比維度       | MLP                      | LSTM                            |
|----------------|--------------------------|---------------------------------|
| 輸入形式       | 靜態特徵快照（94維）     | 時間序列（20×47=940維序列）     |
| 時序資訊       | 無（特徵工程間接編碼）   | 直接學習序列中的時序依賴        |
| 「記憶」機制   | 無                       | Cell State 可記住 20+ 步前資訊  |
| 適合場景       | 統計特徵良好設計時       | 原始序列資料、序列模式重要時    |

#### 優缺點

| 優點                              | 缺點                                         |
|-----------------------------------|----------------------------------------------|
| 原生處理時序依賴關係               | 訓練慢（序列處理不能完全並行化）             |
| Cell State 防止梯度消失            | Catastrophic Forgetting（遺忘早期規律）      |
| 可生成 Saliency Map 解釋哪期最重要 | 超參數更多（seq_len、hidden、layers）        |
| 適合捕捉跨期共現模式               | 若序列本身無統計規律，LSTM 不優於 MLP        |
